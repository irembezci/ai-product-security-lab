# Project 01 — FinBot Security Analysis

## Overview

This project documents a structured AI Product Security analysis of OWASP FinBot, an intentionally vulnerable agentic AI platform developed by the OWASP GenAI Security Project.

The objective is to evaluate the platform from the perspective of an AI Product Security Analyst by applying a systematic security analysis methodology.

The project focuses on understanding the product, identifying critical assets, analyzing trust boundaries, assessing risks and developing security recommendations.

## Target System

### OWASP FinBot CTF

OWASP FinBot is an intentionally vulnerable multi-agent AI platform designed for security education, research and hands-on learning.

The platform simulates a financial services environment where AI agents perform business operations such as:

* Vendor onboarding
* Invoice processing
* Payment operations
* Fraud investigations
* Vendor communications

Project Repository: https://github.com/GenAI-Security-Project/finbot-ctf

All product ownership, source code, trademarks and intellectual property belong to the OWASP GenAI Security Project and its contributors.


## Project Scope

The purpose of this project is not to perform penetration testing or exploit development. Instead, the focus is on conducting a structured AI Product Security analysis to better understand the platform's architecture, business workflows, AI components and security risks.

## Methodology

This project follows the following workflow:

### Phase 1 — Product Understanding

* Product Discovery
* Architecture Analysis

### Phase 2 — Security Analysis

* Asset Inventory
* Trust Boundary Analysis
* Threat Modeling

### Phase 3 — Risk Evaluation

* Risk Assessment
* Security Requirements
* Security Recommendations
